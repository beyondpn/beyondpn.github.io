git subtree & git submodule

2020-06-28 1 min read

git subtree 和 submodule 都是在 git 中引用其他项目的方法，该如何使用呢？这里简要记录一下。

最近在使用 Gridea 写 blog，好的一点是不需要搭建环境，安装，就集成了全部的东西。Gridea 跟 jekyll 的不同之处在于，Gridea 是把生成的 html 文档保存到 xxx.github.io 库，而不是源文件。所以我新建了一个 Gridea 作为源文件的库。

这里现在牵涉到两个库，且有固定的目录结构。即 Gridea/output。
起初我把 output 目录作为一个 submodule 引入到 Gridea 工程中，碰到的问题就是提交的时候比较麻烦。在 Gridea 目录下使用 git status 不能显示 output 的变更，且提交无效。后改用 subtree 解决这一问题。

这里做一个总结。

先说相同点：两者都适用于引入另外一个库的情况，都可以指定分支，tag 引入。
